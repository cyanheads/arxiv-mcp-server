# 0.1.13 — 2026-04-27

Maintenance release with substantive arXiv-side improvements bundled in. Picks up six versions of `@cyanheads/mcp-ts-core` (0.6.17 → 0.7.5), fixes a long-latent category-leakage bug in `arxiv_search`, adds category validation with near-match suggestions, and ships LaTeXML noise stripping in `arxiv_read_paper` for typical 3-4× content shrinkage on math-heavy papers.

## Added

- **`arxiv_search` rejects unknown categories with near-match suggestions** — validates `category` against the static taxonomy before sending the request. On miss, returns a `ValidationError` with prefix-matched suggestions when the archive is valid (e.g. `cs.INVALID` → `cs.AI, cs.AR, cs.CC, cs.CE, cs.CG`) or Levenshtein-distance fallback when the archive itself is unknown. New exports `VALID_CATEGORY_CODES` and `suggestCategories()` in `src/services/arxiv/categories.ts`.
- **`arxiv_read_paper` strips LaTeXML class/id noise from rendered HTML** — typical 3-4× shrinkage on math-heavy papers, no content loss. arXiv's HTML rendering ships `class="ltx_..."` and generated `id="..."` on nearly every element; neither carries information a reader benefits from. Field-tested on `2410.17954`: 161,088 → 109,230 chars (33% shrinkage) with all `<math>`, section structure, and prose preserved.
- **`body_characters` output field on `arxiv_read_paper`** — character count of the cleaned body HTML, distinct from `total_characters` (raw, pre-strip). `max_characters` truncation now applies to the cleaned content, so the budget buys real paper text instead of LaTeXML wrapper attributes.
- **`.env.example` documents `MCP_ALLOWED_ORIGINS`** — required after framework 0.7.1 made the Origin guard fail-closed for remote browser origins. Loopback-only when unset; set to `*` to disable Origin validation entirely.

## Fixed

- **`arxiv_search` no longer leaks results across categories on multi-word queries** — `searchQuery` now wraps the user query in parens before appending `AND cat:{category}`. Previously, `mixture of experts AND cat:cs.CL` parsed under arXiv's grammar as `mixture ∧ of ∧ (experts AND cat:cs.CL)`, so earlier bare terms matched any category. After the fix, `(mixture of experts) AND cat:cs.CL` correctly scopes the category to the entire expression. Field-test verified: returned `ExpertFlow: Efficient Mixture-of-Experts Inference...` with categories `["cs.AI", "cs.CL"]`.
