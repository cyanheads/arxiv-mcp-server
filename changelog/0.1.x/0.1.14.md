# 0.1.14 — 2026-04-30

Maintenance + remediation release. Picks up nine versions of `@cyanheads/mcp-ts-core` (0.7.5 → 0.8.7), adopts the framework's typed error contracts on every tool and resource, replaces the home-grown `withRetry` + `TransientError` pattern with the framework's `withRetry` + `httpErrorFromResponse` utilities, and ships a full-pass remediation for [#8](https://github.com/cyanheads/arxiv-mcp-server/issues/8) (rate-limit retry storm causing 100+ second hangs). `arxiv_get_metadata` now returns the framework's typed `partialResult` shape (wire change — see Migration).

## Added

- **Typed error contracts on every tool and resource** — `errors: [{ reason, code, when, recovery }]` declared on `arxiv_search` (`unknown_category`), `arxiv_get_metadata` (`no_match`), `arxiv_read_paper` (`no_match`, `html_unavailable`), and the `arxiv://paper/{paperId}` resource (`no_match`). Handlers throw via `ctx.fail(reason, …)` and spread `ctx.recoveryFor(reason)` into `data`, so the contract `recovery` string flows to the wire as `data.recovery.hint` and is mirrored into `content[]` text by the framework's error-path parity. Service-layer throws (`ArxivService.search` for `unknown_category`, `ArxivService.readContent` for `html_unavailable`) carry `data: { reason, ...ctx.recoveryFor(reason) }` through the auto-classifier so clients see the same `error.data.reason` they'd see from `ctx.fail`.
- **`partialResult` adoption on `arxiv_get_metadata`** — output now uses `partialResultSchema({ succeededKey: 'papers', failedKey: 'not_found', idKey: 'id', reason: z.enum(['not_in_arxiv']) })`. Adds `totalSucceeded`; replaces `not_found: string[]` with `not_found: [{ id, reason, detail? }]`. See Migration.
- **Server-side `Retry-After` cooldown** — when arXiv returns HTTP 429 with `Retry-After` (or 200 OK with `Rate exceeded.` body), the value is parsed (RFC 9110 §10.2.3 — supports both delta-seconds and HTTP-date), capped at 30s, and applied to every subsequent queued API call via a private `cooldownUntilMs` field. A single throttle event no longer triggers N parallel rate-limit failures across queued callers.
- **User-Agent header on every fetch** — `arxiv-mcp-server (+https://github.com/cyanheads/arxiv-mcp-server)`. arXiv community convention (mirrors the `arxiv.py/<version>` pattern); lets arXiv operators identify and contact maintainers if a client misbehaves. Plus an `Accept` header (`application/atom+xml, application/xml` for API; `text/html` for content fetches).
- **Queue-level cancellation** — `enqueueApiCall(signal, fn)` checks `signal.aborted` at the queue head and skips the task entirely (no fetch, no post-delay), so a cancelled request doesn't consume a 3-second slot. Cooldown waits use a new `abortableSleep` so a cancelled request bails immediately rather than blocking up to 30s.
- **Three new tests** — User-Agent presence, Retry-After cooldown propagation across queued calls (fake timers), pre-cancelled requests skipped at queue head.
- **`scripts/split-changelog.ts`** — picked up from the framework script sync.
- **`start` script in `package.json`** — bare `node dist/index.js` alongside the explicit `start:stdio` / `start:http` variants. External MCP runners and hosted environments that assume the npm-canonical `start` script now work out of the box (template change from framework 0.7.6).

## Changed

- **Fail fast on rate-limit responses ([#8](https://github.com/cyanheads/arxiv-mcp-server/issues/8))** — custom `isArxivTransient` predicate on every `withRetry` call excludes `RateLimited` from the framework's default transient set. HTTP 429 and 200 OK with `Rate exceeded.` body now surface to the caller in <1s instead of cascading through retries. Issue's reproducible 108s/126s hangs reduced to ≤1s for rate-limit and ≤37s worst-case for genuine 5xx blips. `maxRetries` reduced from default 3 to 1 across all `withRetry` call sites.
- **`@cyanheads/mcp-ts-core`** — `^0.7.5` → `^0.8.7`. Nine versions: `maintenance` Phase C script enumeration ([#69](https://github.com/cyanheads/mcp-ts-core/issues/69)) + template `start` script (0.7.6); typed error contracts (`errors[]` + `ctx.fail`) + `httpErrorFromResponse` + `partialResult` + three new error factories + handler-body lints (0.8.0); service-thrown contract reasons + field-test helper hardening (0.8.1); landing-page UX + `maintenance` v2.0 hard-rule on framework adoption (0.8.2); `structuredContent.error` parity with `content[]` + `_meta.error` removed (0.8.3); `ErrorContract.recovery` required ≥5 words (0.8.4); `ctx.recoveryFor(reason)` opt-in resolver + `MCP_AUTH_MODE=none` defaults `tenantId` to `'default'` (0.8.5); `dev:stdio`/`dev:http` scripts removed from templates (0.8.6); `dev` script removed + field-test helper namespacing fix (0.8.7).
- **`ArxivService` retry plumbing migrated to framework utilities** — local `withRetry` and `TransientError` class deleted in favor of `withRetry`/`httpErrorFromResponse` from `@cyanheads/mcp-ts-core/utils`. HTTP error classification now uses the framework factories (`rateLimited`, `serviceUnavailable`, `invalidRequest`, `serializationError`); HTTP 5xx responses to arxiv.org/html and ar5iv route through `httpErrorFromResponse` with a `codeOverride` mapping all 5xx to `ServiceUnavailable` for retry consistency.
- **`PaperLookupResult.not_found` → `PaperLookupResult.not_found_ids`** — internal service shape renamed for clarity. Tool layer wraps it into the framework's `partialResult` wire shape; resource layer is unchanged.
- **`CLAUDE.md` Errors section rewritten to lead with typed contracts** — factories demoted to fallback. Added sections on service-thrown reasons, `httpErrorFromResponse`, and `withRetry` adoption. Domain Notes now document the rate-limit fail-fast policy. Tool/Resource pattern examples updated with `errors[]` + `ctx.fail`. Commands table drops the removed `dev:stdio`/`dev:http` rows.
- **Skills synced from package** — `add-service` 1.3 → 1.5, `add-tool` 1.8 → 2.4, `api-context` 1.1 → 1.2, `api-errors` 1.0 → 1.4, `design-mcp-server` 2.7 → 2.8, `field-test` 2.0 → 2.3, `maintenance` 1.6 → 2.0, `release-and-publish` 2.1 → 2.2, `report-issue-framework` 1.3 → 1.4, `security-pass` 1.1 → 1.2, `setup` 1.5 → 1.6.

## Removed

- **`dev:stdio` and `dev:http` scripts from `package.json`** — adopted from framework 0.8.6/0.8.7 template cleanup. Smoke-test path is now `bun run rebuild && bun run start:stdio` (or `start:http`), giving every developer and smoke-test path the same execution surface as production.
- **Local `withRetry` and `TransientError` class in `ArxivService`** — superseded by framework `withRetry` from `@cyanheads/mcp-ts-core/utils`.

## Migration

`arxiv_get_metadata` output shape changed:

| Before | After |
|:-------|:------|
| `{ papers, not_found?: string[] }` | `{ papers, totalSucceeded, not_found?: [{ id, reason: 'not_in_arxiv', detail? }] }` |

Clients that read `result.structuredContent.not_found[i]` as a string need to read `result.structuredContent.not_found[i].id` instead. The `format()` text representation is unchanged for the eyeballs-on case (still says `Not found: <id>`).

Tool error envelopes now carry `result.structuredContent.error.{code, message, data?}` instead of the framework's previous `result._meta.error.{code, data?}` shape (this surface was removed in `mcp-ts-core` 0.8.3). No consumer code in this server referenced `_meta.error`.
