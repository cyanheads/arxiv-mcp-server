# 0.1.15 — 2026-04-30

UX improvements to `arxiv_read_paper` driven by a tool-verification handoff that flagged the absence of pagination and ambiguous fallback wording. Long papers can now be read chunk-by-chunk via a `start` character offset instead of forcing a re-fetch with a higher `max_characters`. The HTML cleaner now preserves section-structure markers so future section-scoped reads stay feasible. No wire-shape break — `start` is an additive optional input with default `0`.

## Added

- **`start` offset on `arxiv_read_paper`** — character offset into the cleaned body where the slice begins. Defaults to `0`. Pairs with `max_characters` to page through papers that exceed the cap (e.g. `start=100000, max_characters=100000` returns chars 100,000–199,999). Output now echoes `start` so callers can confirm the slice range. The `truncated` flag's semantics shifted from "content was clipped" to "more body content exists past this slice"; format text emits four distinct notices: top-of-paper truncation, mid-paper truncation (with `start={next}` hint), final-chunk completion, and past-end recovery (`start ≥ body_characters`). Service signature `readContent(paperId, options, ctx)` now takes a `ReadContentOptions { maxCharacters?, start? }` object — internal change, no external callers.
- **Section-structure markers preserved by the HTML cleaner** — `stripLatexmlNoise` now keeps `class="ltx_..."` attributes that contain `ltx_section`, `ltx_subsection`, `ltx_subsubsection`, `ltx_paragraph`, `ltx_subparagraph`, `ltx_appendix`, `ltx_bibliography`, `ltx_abstract`, `ltx_acknowledgement[s]`, `ltx_title`, `ltx_part`, or `ltx_chapter`. Decorative classes (`ltx_text`, `ltx_font_*`, `ltx_para`, `ltx_p`, etc.) still strip out. Cost: ~1.7% increase in `body_characters` on math-heavy papers (107,052 vs 105,214 chars on `1706.03762`); benefit: section navigation stays feasible for downstream tooling without a re-fetch.
- **Six new tests** — `start`-passthrough at the tool layer; mid-paper, final-chunk, and past-end format notices; pagination, exact-end-boundary, and past-end behavior at the service layer; structural-marker preservation in the cleaner.

## Changed

- **`arxiv_read_paper` description rewritten** — explicit about the fallback chain (arxiv.org/html → ar5iv.labs.arxiv.org). PDF-only papers return `html_unavailable` with `pdf_url` for direct download — the tool does not auto-extract PDF content. Pagination workflow signposted in the description.
- **`html_unavailable` error `when` text** sharpened: "Paper exists but neither arxiv.org/html nor ar5iv has an HTML rendering; only the PDF is available."
