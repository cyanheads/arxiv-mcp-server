# 0.1.12 — 2026-04-24

Maintenance release. Picks up the `@cyanheads/mcp-ts-core` 0.6.17 HTTP transport hardening and adopts three upstream CLAUDE.md template improvements surfaced during the `maintenance` skill pass. No runtime code changed.

## Changed

- **`@cyanheads/mcp-ts-core`** — `^0.6.16` → `^0.6.17`. Internal HTTP transport fixes: per-server notifier race resolved (a concurrent HTTP request could previously cross-wire `notifyResourceListChanged` / `notifyResourceUpdated` onto another request's server), per-request close path converted to parallel awaited close under a 5s bounded timeout, and three new metrics (`mcp.http.close_failures`, `mcp.http.per_request.created`, `mcp.http.per_request.finalized`) for attributing HTTP-mode retention. Consumer-visible API unchanged. Transitive `hono` `4.12.14` → `4.12.15`.
- **`CLAUDE.md`** — "What's Next?" inserts `security-pass` at step #8, renumbering `polish-docs-meta` and `maintenance` to #9/#10. Checklist now describes `format()` client surface differences explicitly (Claude Code reads `structuredContent`, Claude Desktop reads `content[]`; both must carry the same data) and adds three items for external API wrapping: raw/domain/output schemas reviewed against real upstream sparsity/nullability, normalization preserves uncertainty (no fabricating facts from missing arXiv fields like `comment` / `journal_ref` / `doi`), and tests include a sparse-payload case.
- **`CLAUDE.md` header** — version corrected `0.1.10` → `0.1.12` (was stale from a prior release).
