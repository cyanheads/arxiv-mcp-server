# 0.1.10 — 2026-04-24

Framework refresh from `@cyanheads/mcp-ts-core` 0.5.3 to 0.6.16 — adopts the directory-based changelog system, fixes new recursive `describe-on-fields` lint warnings, and syncs skills + scripts to match the upstream package.

## Added

- **Directory-based changelog** — migrated monolithic `CHANGELOG.md` into per-version files under `changelog/0.1.x/` with frontmatter (`summary`, `breaking`); `CHANGELOG.md` is now an auto-generated navigation index. New `changelog:build` / `changelog:check` scripts in `package.json`. Framework pattern from mcp-ts-core 0.6.0.
- **Framework scripts** — added `scripts/build-changelog.ts`, `scripts/check-docs-sync.ts`, and `scripts/check-skills-sync.ts` from the package (Phase C of the updated `maintenance` skill in 0.6.16). Refreshed `scripts/devcheck.ts` and `scripts/tree.ts` to pick up the new `Docs Sync`, `Skills Sync`, and `Changelog Sync` devcheck steps.
- **3 new skills** synced from framework: `api-linter` (definition lint rule reference, v0.6.0), `release-and-publish` (post-wrapup ship workflow, v0.6.10), `security-pass` (8-axis MCP server audit, v0.6.14).

## Changed

- Updated `@cyanheads/mcp-ts-core` from `^0.5.3` to `^0.6.16`
- Updated `@biomejs/biome` from `^2.4.12` to `^2.4.13`
- Updated `vitest` from `^4.1.4` to `^4.1.5`
- Synced 15 existing project skills to framework 0.6.16 (`add-app-tool`, `add-prompt`, `add-resource`, `add-service`, `add-tool`, `api-context`, `api-services`, `api-utils`, `design-mcp-server`, `field-test`, `maintenance`, `polish-docs-meta`, `report-issue-framework`, `report-issue-local`, `setup`) and refreshed `.claude/skills/` mirror
- `CLAUDE.md` — added `api-linter`, `release-and-publish`, `security-pass` to skills table; Publishing section now references the `release-and-publish` skill

## Fixed

- Five new `describe-on-fields` lint warnings surfaced by mcp-ts-core 0.6.16's recursive linter (previously top-level-only):
  - Added object-level `.describe()` on `PaperMetadataSchema` — covers `output.papers[]` on both `arxiv_search` and `arxiv_get_metadata`
  - Added object-level `.describe()` on `CategorySchema` — covers `output.categories[]` on `arxiv_list_categories`
  - Added per-variant `.describe()` on both union variants of `arxiv_get_metadata` `paper_ids` input (single ID string and array)
